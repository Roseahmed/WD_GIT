
Different stages of git Version Control:-
a)current working directory/space.
b)staging area.
c)Local Repository(.git).
d)Remote Repository(github).


/*Process for creating git version Control

git init; // Initialize git.
git add filename; // Add files into staging area.
git status; // Check the status of files.
git commit -m "commit message better to give in present tense"; // Commit the files.
git diff fileName; // Check the changes made in files.
         OR 
git show commit_id; // Check changes made in the files.
git log; // Check all commits details.
git log --oneline; // Check commits in one line.
*/



/*Ignore files in Current Working for git version Control

1.create hidden file: .gitignore;
2.open the file and write which files/directory type you want to ignore eg:
*.cpp; 
*.c;  // it will ignore all the c and cpp files type.
3.Add the file in staging area.
4.Commit the file.
*/


/*Process for creating branch

git branch; // Check the total no of branches and current/working branch.
git branch newBranchName; // Create new branch.
git checkout branchName; // Switch branch.
N.B:-New file creation or modification in branch will also reflect in master branch until it is added in staging area and committed.
*/


/*Process for merging different branches in master branch

1.First go the branch to which you want merge with, usually master branch.
2.git merge mergeBranchName.
         
         **Merge Conflict**
when same file having different content in different branches if you do merge conflict occurs.

solution:Open the file and edit the content and commit the file.
*/


/*Process for adding tag to commit

git tag -a tagname -m "tagMessage" commit_id; // Add tag name to commits.
git tag; // Check tag name.
git tag -d tagname; // Remove tag.
N.B:- Tag is alternative for commit_id.
*/


/*Process for removing file from workspace, staging area and commit

git restore fileName; // Remove files from working space.
git restore --staged fileName; // Remove file from staging area.
git reset --mixed HEAD~1; // Remove the last one commit and put it working space, (default reset).
git reset --soft HEAD~2; // Remove the last two commit and put it in staging area.
git reset --hard HEAD~3; // Remove the last three commit completely from all three stages.
N.B:-Use reset for local repository only, reset command is not recommended for public or remote repository(github).
*/


/*Process for reverting previous commit

1.git revert commit_id; // Revert previous commit
2.It will automatically open nano editor.
3.Add the commitMessage and save the commit and exit.
*/


/*Process for deleting untracked files 

git clean -n; // Remove all the untracked files/directory that are not ignored.
git clean -f;
*/


/*Process for moving any work to stash

N.B:- For moving files in stash the file should be at least committed once.
git stash; // Move the modification files from working area to stash.
git stash list; // Display the list and list_id of stash file.
git stash apply list_id; // Bring back the data/files to working directory.
git stash clear; // Clear the item copied in stash area.
*/ 


/*Process for pushing Local Repository to github

git remote add origin [anyname] link_of_github_Repository; 
git branch -M main/master;
git push -u origin main/master;
*/

/*Process for pulling code from remote Repository(github)

git remote add origin Link_of_github_Repository.
git pull origin master.
*/

/*Process for cloning github Repository to local directory
git clone Link_Of_Repository.
*/
